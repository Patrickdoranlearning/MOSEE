## Methodology — the investment committee's verdict

Consensus across Buffett, Munger, Fisher, and Lopez de Prado, each grounded in the code:

### The Doubler screen is currently an adverse-selection machine
- `implied_annual_return = (composite_base/price)^(1/5) − 1` is **multiple-convergence, not growth**. It is maximized when the model disagrees most with the market — and absent validation, that ranks stocks by *model error*. Value traps congregate exactly in the 15–40% band (Munger).
- Default gates are loose: `MIN_CONFIDENCE = 50`, `MAX_MOS = 1.0` (no margin required), **no quality floor, no verdict filter** — cigar butts rank first (Buffett).
- Fix direction: hard-gate `quality_score ≥ 65` + verdicts BUY/STRONG BUY/ACCUMULATE by default, and add a **growth leg** (sustainable EPS growth × terminal multiple) so doublers can qualify on compounding, not just cheapness (Fisher).

### The tool penalizes the compounders it's hunting
- Buffett lens scores **one year** of ROE/ROIC — no 10-year consistency, no incremental ROIC, no share-count trend. `calculate_roe_history` exists in `indicators.py` but is never used by the lens.
- `fundamental_data.py:1141` uses **average capex as maintenance capex** → owner earnings understated for reinvesting growers; growth capped at 15% (`fundamental_data.py:840`) overstates PEG for fast growers.
- Structural caps (MAX_PE 40/50, perpetuity growth 6%, price ≤ 0.7× conservative) mean **no great compounder of the last 30 years would ever pass** — "the tool will tell you to sell Costco, forever" (Munger).

### Missing data is fabricated into signal
- `earnings_growth` defaults to 0.05, `roe` to 0.10, `discount_rate` to 0.10 (`valuation_range.py:706-727`); missing debt/equity defaults to 1.0 and **awards points** in the Buffett lens (`mosee_intelligence.py:251`); missing current liabilities defaults to 1 (`run_weekly_analysis.py:301`). Unknown ≠ average. Should return None and lower confidence.

### Confidence measures agreement, not correctness
- `confidence.py` rewards low CV across valuation methods — but all five lenses chew the same yfinance trailing data. Five correlated models agreeing is not evidence. Highest-conviction output is currently the least trustworthy.

### Zero validation evidence (Lopez de Prado)
- Lens weights are hand-picked round numbers; no backtest harness, no benchmark, no out-of-sample anything. Yahoo extended history is survivorship-biased — **do not backtest on it**.
- The fix is **prospective point-in-time logging**, starting now (infrastructure already exists in `mosee_stock_analyses`): freeze weights, treat weekly runs as immutable predictions, join to +3/6/12-month realized returns vs SPY, compute Spearman IC at 12 months. Until then the score is decoration.

### Verdict churn pushes over-trading
- Binary cliffs (quality ≥ 65, MoS 0.69 vs 0.71) + weekly re-scoring + "Consider initiating" action items = action bias re-administered every Monday. Add **hysteresis** (no verdict flip inside the noise band) and show the previous verdict beside the current one.

### Inconsistent math reaching Patrick
- PAD/DCF in `fundamental_analysis/valuation.py:26,131` discounts at the 4% risk-free rate (no equity risk premium) while the engine uses 10% (`valuation_range.py:707`) — two contradictory valuations for the same stock.
- `industry_pe` hardcoded to 15 for all industries (`run_weekly_analysis.py:490`); book-value method gets HIGH confidence (1.5×) whenever quality ≥ 60 — nearly meaningless for asset-light businesses.
- Cyclical `required_mos = 0.60` but verdict rationale text hardcodes 0.70/0.50 (`mosee_intelligence.py:704-707` vs `:589,604`).

### Patrick's rules aren't in the repo
- Only the Doubler target (100%/5yr = 14.87% CAGR) is codified. "Rules in your head bend; rules in code don't" (Buffett). A `RULES.md` (or rules engine) should gate verdicts/screens.

## UX — from data viewer to decision tool

1. **No "what changed"** — biggest break in the weekly journey. History exists in the DB; no screen diffs two runs. Build a weekly digest: verdict changes, new Doubler entrants, prices crossing buy-below.
2. **Four tables, three ranking philosophies** — same stock glows green on one page, gray on another (`StockLookup` conservative min-of-methods vs `StockTable`/`FeaturedStockCard` optimistic `pad ?? dcf ?? book`, different thresholds). One canonical conviction metric (implied return × confidence band) everywhere.
3. **Naked numbers** — list views show point estimates to 3 decimals with no confidence/range; the well-built range UI is buried in the stock page.
4. **Buy decision dead-ends** — Action Items checkboxes don't persist; no watchlist, decision journal, or position link ("you own this; verdict downgraded since you bought").
5. The **screener WIP is the most decision-shaped page in the app** — it (or the digest) should be the landing experience. Dead code in `PicksFilter.tsx` (`applyQuickFilter`).

Top product features by decision impact: weekly diff digest → actionable-now list (price ≤ buy_below + confidence) → watchlist + persisted research checklist/journal → holdings↔analysis link → Fisher "15 Points Worksheet" per candidate (pre-filled where data exists, unanswered questions listed for hand-scuttlebutt).

## How Patrick gets Buffett-like returns with this tool (committee synthesis)

- **The screen says "no", never "yes"** — passing makes a candidate; conviction comes from hand-reading (3 annual reports, who-loses-if-they-vanish) supported by the worksheet.
- **Punch-card discipline**: 5–10 positions, sized by downside-if-wrong; `buy_below_price` is the limit order; otherwise sit on hands. Never override the screen to buy — only to pass.
- **Codify the rules** so they can't bend in a drawdown.
- **Start the evidence clock now**: freeze weights, log predictions point-in-time, measure IC vs SPY at 12 months. Don't touch weights until then.
